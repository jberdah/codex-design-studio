# OpenAI Build Week demo — 2:45 target

Use the packaged application and the owned fictional brand prepared for the submission. Do not use third-party trademarks or copyrighted design assets in the final recording.

## 0:00–0:15 — The hook

Show the desktop workspace and say:

> Every company already has a brand, but its evidence is scattered. Codex Design Studio turns that evidence into one executable Design System, then uses GPT-5.6 and Codex to create and verify every new asset.

## 0:15–0:50 — Evidence-aware project bootstrap

Create the fictional project from a concise brief and one reference website owned for the demo. Show:

- Extract versus Inspire intent;
- the non-blocking responsibility warning;
- facts, inferences, assumptions, and at most three questions;
- one source observation that materially changes the proposed palette or typography; and
- explicit approval before the project becomes active.

Explain that raw onboarding text is transformed into a structured creative brief rather than copied into a template.

## 0:50–1:10 — One living Design System

Open the Design System and point to the approved evidence, tokens, and version. Briefly show that the same project contains a Web artifact and an editable presentation.

Say:

> GPT-5.6 reconciles the creative evidence. The Design System becomes the shared source of truth.

## 1:10–1:50 — Codex changes the real artifact

Open the landing page, briefly toggle **Edit canvas** to demonstrate direct manipulation, then select the hero or navigation in the live preview and submit one composition-level instruction prepared in advance.

Show that the resulting HTML/CSS/SVG visibly changes—not only the copy or colours. If a live call is too slow, use the rehearsed deterministic candidate while explaining that the same transaction boundary is exercised.

Say:

> Codex edits the actual project files inside a scoped workspace. It is not returning a screenshot or a design description.

## 1:50–2:15 — Proof before promotion

Compare the current artifact and candidate. Show desktop/mobile renders, pixel difference, overflow status, and the explicit Accept/Reject choice. Mention that conservative checks keep the proposal separate instead of silently discarding it or trusting the model.

Say:

> Codex Design Studio independently renders and checks the result. The user remains the final authority.

## 2:15–2:35 — One system, several outputs

Open the editable slide canvas, show a generated visual asset version, then export the responsive HTML and editable PPTX. Keep this fast: the point is shared brand consistency, not a tour of every control.

## 2:35–2:45 — Close

Return to the full workspace and say:

> GPT-5.6 reasons and creates. Codex acts. Codex Design Studio verifies. The user decides.

## Recording checklist

- final duration below 3:00;
- 1920×1080 capture with cursor visible;
- spoken audio plus burned-in English captions;
- no notifications, credentials, local absolute paths, or third-party trademarks;
- fictional project reset before recording;
- downloads and live network calls rehearsed before the take;
- final public YouTube link tested in a private browser window.
